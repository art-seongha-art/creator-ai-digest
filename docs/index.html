Comfy MCP: 에이전트가 ComfyUI 이미지·영상 워크플로우를 자동 실행하는 방향

Comfy MCP는 AI 에이전트가 ComfyUI 기반 이미지·영상 생성 워크플로우를 호출하고 자동화할 수 있게 하는 흐름으로 소개됐습니다. Comfy Cloud를 사용하면 낮은 사양 PC에서도 복잡한 노드 작업을 에이전트 지시로 반복 실행하는 제작 파이프라인을 구성할 수 있다는 점이 핵심입니다.

인사이트: 디자이너는 ‘프롬프트 작성→노드 실행→결과 선별→변형안 생성’을 작은 자동화 과제로 나눠볼 수 있습니다. 수업에서는 학생들이 직접 노드를 모두 외우기보다, 워크플로우를 설계하고 에이전트에 맡길 부분을 정하는 실습으로 바꾸기 좋습니다.

PhysisForcing: 물리적으로 그럴듯한 로봇 월드 시뮬레이터 연구

PhysisForcing은 비디오 생성 모델을 로봇 조작용 월드 시뮬레이터로 쓸 때 생기는 문제, 예를 들어 물체가 갑자기 순간이동하거나 로봇-물체 접촉이 어색해지는 현상을 줄이려는 연구입니다. 생성 영상에 물리적 일관성을 더해, 로봇과 인터랙티브 장면을 미리 테스트하는 방향을 보여줍니다.

인사이트: 교육 현장에서는 ‘AI가 만든 장면이 보기에는 자연스러워도 실제 동작 가능한가?’를 토론하기 좋은 사례입니다. 게임/실감미디어 제작자는 영상 생성 결과를 프리비즈로 쓸 때 물리 검증 단계가 필요하다는 체크리스트로 활용할 수 있습니다.

Meta Muse Image가 인스타그램·WhatsApp의 AI 이미지 생성에 들어갔습니다

Meta가 Superintelligence Labs의 첫 이미지 생성 모델 Muse Image를 공개하고 Meta AI, Instagram, WhatsApp의 이미지 제작 기능에 적용하기 시작했습니다. 다른 사용자를 AI 사진 장면에 불러오는 식의 소셜 이미지 제작 흐름까지 포함되어, 이미지 생성이 ‘툴 안에서 따로 만드는 작업’보다 플랫폼 내부 협업 기능에 가까워지고 있습니다.

인사이트: 창작자는 SNS용 콘셉트 시안, 캠페인 무드보드, 캐릭터 협업 컷을 더 빠르게 실험할 수 있습니다. 다만 실제 인물/계정이 얽히는 기능은 동의와 초상권 안내를 워크숍 규칙에 먼저 넣는 것이 좋습니다.

Qwen-Image-2.0-RL: 이미지 생성 모델을 RLHF로 더 정교하게 맞추는 기술 보고서

Qwen-Image-2.0-RL은 확산 이미지 모델의 시각 품질과 지시문 이행 능력을 높이기 위해 RLHF와 on-policy distillation을 적용한 후학습 파이프라인입니다. 보상 모델을 과제별로 구성해, ‘프롬프트를 얼마나 잘 따랐는지’와 ‘이미지가 얼마나 안정적인지’를 함께 개선하려는 접근입니다.

인사이트: 이미지 생성 수업에서는 같은 프롬프트라도 모델 후학습 방식에 따라 결과 해석과 수정 횟수가 달라진다는 점을 설명하기 좋습니다. 실무자는 브랜드 가이드나 반복 스타일을 맞추는 파인튜닝/후처리 전략의 참고 사례로 볼 수 있습니다.

Seedance 2.5: 30초 영상, 다중 레퍼런스, 3D 프리비즈를 강조한 영상 생성 업데이트
