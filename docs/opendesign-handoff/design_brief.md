# OpenDesign 리디자인 요청서

## 프로젝트

**창작자를 위한 AI 다이제스트**

매일 발행되는 공개 웹진형 AI 다이제스트입니다. 독자는 예술·디자인·교육·콘텐츠 현장의 창작자, 교육자, 디자이너, 기획자, 연구자입니다.

## 발행 정책

- 매일 발행합니다.
- 상단에는 기간 범위를 쓰지 않고 **발행일만** 표시합니다.
- 수집/선정 기준은 **발행일 기준 최근 10일 안의 주요 토픽**입니다.
- 특정 개인을 대상으로 한 표현은 넣지 않습니다.
- “성하님”, “박성하”, “학과 전용”처럼 개인/특정 집단으로 좁히는 표현은 금지합니다.

## 리디자인 목표

현재 HTML을 바탕으로, **흰색 바탕의 깔끔한 웹진 타입**으로 리디자인해주세요.

원하는 느낌:

- white background
- clean editorial web magazine
- readable, friendly, practical
- 과한 SaaS 대시보드 느낌 금지
- 과한 gradient/glassmorphism 금지
- 카드만 반복되는 구조보다 기사형 리듬이 있는 레이아웃
- 이미지/영상이 자연스럽게 보이는 매거진 레이아웃
- 모바일에서도 읽기 쉬운 구조

## 문체 방향

- 어렵고 추상적인 AI 뉴스 해설보다, 현장에서 바로 이해할 수 있는 설명
- 독자가 다음 작업에 바로 연결할 수 있게
- 친근하지만 가볍지 않게
- 전문용어는 필요하면 쉽게 풀어서 설명

## 페이지 구조 제안

1. Header
   - 제목: 창작자를 위한 AI 다이제스트
   - 발행일만 표기: 예) 발행일 2026.06.17
   - 별도의 시스템 설명 문구나 선정 기준 문장은 넣지 않습니다.

2. Lead story
   - 가장 시각적/실용적 항목 1개를 크게 배치
   - 이번 예시는 NVIDIA MotionBricks가 lead story로 적합
   - 영상이 있으면 video 영역을 크게

3. Article list
   - 나머지 항목은 2-column 또는 editorial card layout
   - 각 항목은 source/date/category/title/summary/insight/link를 포함
   - 이미지가 있으면 썸네일, 없으면 심플한 타입 기반 placeholder

4. Footer
   - 생략하거나, 원문 링크/크레딧처럼 꼭 필요한 정보만 최소화합니다.
   - 운영 설명 문구는 넣지 않습니다.

## 금지 사항

- 특정 개인 이름/호칭 노출 금지
- 내부 파이프라인, 후보 수, 에이전트, 검수, local path 노출 금지
- “OpenDesign 요청서” 같은 제작 과정 문구를 최종 페이지에 보이게 하지 말 것
- 임의의 가짜 통계/수치 추가 금지
- 원문에 없는 사실 추가 금지

## 제공 파일

- `source_current.html`: 현재 공개 HTML
- `source_content.json`: 현재 콘텐츠 JSON
- `assets/`: 현재 이미지 assets

## 산출물 요청

- `index.html` 하나로 열리는 완성형 HTML
- CSS는 내장해도 좋고, 필요하면 별도 파일로 분리해도 됩니다.
- 최종 페이지는 GitHub Pages의 `docs/`에 반영하기 쉬운 구조로 만들어주세요.

## 추가 수정 원칙

- 선정 기준과 운영 정책은 내부 원칙으로만 사용하고, 독자에게 보이는 화면은 기사/토픽 중심으로 구성합니다.
- 헤더 아래 설명문과 footer 운영 설명은 넣지 않습니다.
